Rapid Framework

Mobile experience, everywhere

How is this different?

This is a basic framework that contains a few elements to get you started with your web project. There are many great frameworks out there, but I thought I would make my own.

This is a mobile first framework, but also a mobile experience framework.

What does 'mobile experience' mean?

If you use the internet on your mobile, you can often get a much better user experience than the desktop version of the same website. Here are a few reasons why:

  • The main content is prioritised to the top of the page. Additional content sections often wrap to the bottom of the page
  • The content is not surrounded by distracting adverts and flash animations
  • The site feels less cluttered and cleaner

So I thought it would be a great idea to make a website that is the same great experience on mobile, but keep the same great experience for tablets and desktops as well.

The GREECE Principles

To summarise this philosophy, I have come up with the GREECE principles...

Gridless freedom

Notice that there is no grid. The content doesn't jump about the page depending what size the screen is and wrap and behave differently on different devices. You get a consistent layout on all devices.

Reduced CSS

Rather than spending ages overwriting default browser behaviour, make use of it. Accept that your site won't look exactly the same in every browser. Style what matters most to you, but leave the rest to default browser behaviour. Write clean CSS that doesn't overwrite other rules. Use standard HTML tags as they are meant to be used.

Easy to read

If you have a larger screen then make the text as easy to read as on the mobile version, but make use of the larger screens space for images and video content.

Easy to use

Make your site easy to use, with clear navigation and calls to action. Make buttons and links as easy to click with a mouse on a desktop device as it is to tap a button on a mobile. Make it obvious what will happen when you click on a button with thoughtful link text (don't use 'Click Here')

Content first

Rather than trying to fill every spare inch of space on the page, declutter the page and only show what you really need to show. Whitespace is nice for a user and helps them concentrate on what really matters.

Easy to code

As well as focusing on making it easy to read and use, make it easy to code.
